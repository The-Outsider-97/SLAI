Understanding the Prediction Table

1h, 24h, 1w Prediction

These values represent the model's forecast for the stock price at different future time horizons (1 hour, 24 hours, and 1 week). Currently, as seen in app.py (get_predictions_api endpoint), these predictions are derived from the finance_agent_instance.adaptive_learner.robust_predict() method. The robust_predict method in adaptive_learning.py provides a single 'prediction' value. In the current implementation, this single prediction value (pred_value) is used for all three time horizons (1h, 24h, 1w). This suggests the model, in its current state as exposed to the API, primarily generates a short-term forecast that is then presented across these intervals. A more advanced system might have distinct models or post-processing steps for each time horizon.

The core prediction comes from an ensemble of models and a meta-model within the AdaptiveLearningSystem. The ensemble models (self.ensemble_models) are simple neural networks. Their outputs are combined using learned prediction_weights. This weighted ensemble prediction (base_pred) is then refined by a meta-model (self.meta_model, an LSTM) that also contributes to uncertainty estimation. The final displayed 'prediction' is a weighted average of the base ensemble prediction and the meta-model's prediction.

ELR Component (Adaptive Learning Rate)

The "ELR Component" refers to the adaptive learning rate used by the AdaptiveLearningSystem. In app.py, this is fetched from diagnostics.get('adaptive_learning_rate', 0.0), which in turn gets this value from adaptive_learner.get_diagnostics().

Looking at adaptive_learning.py (update method), the learning rate is dynamically adjusted: self.learning_rate = max(1e-5, self.learning_rate * self.lr_decay / (1 + recent_vol)). This means the learning rate (self.learning_rate) is influenced by its previous value, a decay factor (self.lr_decay), and the recent volatility of prediction errors (recent_vol). A higher recent error volatility tends to decrease the learning rate, making the system more cautious in its updates. The "ELR Component" likely reflects this dynamically adjusted learning rate.

Sentiment Impact

This value quantifies the influence of current market sentiment on the stock's outlook. In app.py (get_predictions_api), it's fetched using finance_agent_instance.trend_analyzer.get_latest_sentiment(symbol).

The CulturalTrendAnalyzer (in cultural_trend_analyzer.py) calculates sentiment via its sentiment_scraper (PublicSentimentScraper). The get_latest_sentiment method returns self.last_sentiment.get(symbol, 0.0), which is updated in the refresh_sentiment method. This suggests that sentiment scores are scraped or calculated (e.g., from news, social media) for a given symbol and stored. The "Sentiment Impact" value directly reflects this most recently calculated sentiment score, typically ranging from -1 (very negative) to +1 (very positive).

Uncertainty

This metric quantifies the model's uncertainty about its prediction. In app.py (get_predictions_api), it's calculated as (1 - confidence_val) * current_price * 0.05. Here, confidence_val comes from prediction_output['confidence'] which is from adaptive_learner.robust_predict().

The robust_predict method in adaptive_learning.py calculates confidence as float(np.exp(-combined_std)), where combined_std is the combined standard deviation from two sources:

  1. meta_std: Standard deviation of predictions from Monte Carlo dropout passes on the meta-model.
  2. hist_vol: Standard deviation of recent prediction errors (self.error_history).
The combined_std is np.sqrt(meta_std**2 + hist_vol**2). A higher combined_std leads to lower confidence. The UI's "Uncertainty" then scales this (1 - confidence) by the current price and a fixed factor (0.05), representing a price range of uncertainty.

p-value

The p-value is a statistical measure often used in hypothesis testing. In this context, it likely relates to the significance of the model's prediction or the factors influencing it. In app.py (get_predictions_api), it's fetched from diagnostics.get('p_value', 0.05), where diagnostics comes from finance_agent_instance.adaptive_learner.get_diagnostics().

However, the AdaptiveLearningSystem.get_diagnostics() method in adaptive_learning.py does not currently return a 'p_value'. It returns: 'current_weights', 'learning_rate', 'recent_error', 'error_volatility', 'drift_detected', 'model_diversity'. The value 0.05 in the API is a default fallback. For this to be a meaningful value from the model, the get_diagnostics method (or another relevant method in AdaptiveLearningSystem) would need to be updated to calculate and return a p-value, possibly related to concept drift detection (e.g., the p-value from the t-test in _detect_drift) or the statistical significance of prediction model parameters/features.

Critical Value

The critical value is typically used in conjunction with a test statistic (like a t-statistic or z-score) and a p-value in hypothesis testing to determine statistical significance. In app.py (get_predictions_api), this is hardcoded as "1.96".

A critical value of 1.96 corresponds to a 95% confidence level in a two-tailed test using a normal distribution (e.g., for constructing a 95% confidence interval). While commonly used, its relevance here depends on the (currently missing) specific statistical test being referenced by the "p-value". If the "p-value" were from a test comparing a model coefficient to zero, and the test statistic exceeded 1.96 (in absolute value), then the p-value would typically be less than 0.05, indicating significance.

Confidence

This value represents the model's confidence in its prediction. In app.py (get_predictions_api), this is directly from prediction_output['confidence'], which is sourced from finance_agent_instance.adaptive_learner.robust_predict().

As explained in the "Uncertainty" section, the robust_predict method in adaptive_learning.py calculates confidence as float(np.exp(-combined_std)). combined_std is derived from the meta-model's prediction variance (via Monte Carlo dropout) and the historical volatility of prediction errors. A lower combined standard deviation (less spread in MC predictions and lower historical error volatility) results in a confidence value closer to 1.0 (higher confidence). Conversely, higher variability leads to confidence closer to 0.0.

Lower and Upper Bounds

These values define a confidence interval around the predicted price. In app.py (get_predictions_api), they are calculated as:

  • Lower Bound: pred_value - ( (1 - confidence_val) * current_price * 0.05 )
  • Upper Bound: pred_value + ( (1 - confidence_val) * current_price * 0.05 )
This is essentially Predicted Price ± Uncertainty_Value_From_UI.

The term ( (1 - confidence_val) * current_price * 0.05 ) is the "Uncertainty" value shown in the UI. The confidence interval is constructed by taking the point prediction (pred_value) and adding/subtracting this calculated uncertainty margin. This interval provides a range within which the actual price is expected to fall, based on the model's current confidence level. A higher confidence (lower uncertainty term) will result in a narrower interval.

Alternatively, a more statistically grounded confidence interval could be derived directly from the combined_std in AdaptiveLearningSystem.robust_predict(), for example: Predicted Price ± (Critical_Value * combined_std_scaled_to_price). The current UI calculation is a more direct transformation of its own "Uncertainty" metric.
